# 需求内容

你是一个精通docker、node.js、cpp的开发工程师，我现在需要你帮我实现一个node.js的编译构建缓存的工作流，编译后应该要保留缓存文件在镜像中，方便我后续添加CPP代码添加模块后编译出最终二进制

要求如下：

1. 支持生成Linux/Windows/macOS的构建缓存镜像，以及各自操作系统的X64、ARM64的架构，以及Linux的musl版本，应该是可以构建出8种镜像（如果你有更好的方案也可以提出）
2. 要求dockerfile只能有一个，从ARG中获取要构建的Node.js版本、操作系统、架构、LIBC版本，然后拉取Node.js代码仓库的代码切到正确的分支并进行对应目标的构建
3. from需要从debian-trixie开始
4. 构建的结果应该是包含源码、编译缓存或中间文件的新docker镜像，用户使用这个镜像时应该很快就能构建出最终的二进制发行版。用户也可以添加任何CPP模块，编译模块后也可以快速构建出最终的二进制发行版
